问题背景 Problem Statement

  • 输出截断风险：大型项目（20+ 文件）依赖单次 LLM 调用生成全部代码， 受上下文窗口限制，常出现尾部文件缺失、代码不完整等问题。
  • 代码浏览体验差：通用代码查看器缺乏文件树、标签页与语法高亮， 用户难以快速导航并审阅生成的服务结构。
  • 生成过程黑盒：用户在生成期间无任何进度反馈，无法判断当前阶段、 剩余工作量，也无法定位出错环节。

技术细节 Technical Details

1
PLAN · 架构规划
基于需求进行复杂度分级（micro / small / medium / large），输出模块拆分、技术选型与文件清单。
2
SPECS · 规格先行
生成 PRD 与 OpenAPI 规范文档，为后续模块生成提供契约约束。
3
TESTS · 测试骨架
TDD 方式先行生成测试用例骨架，锁定模块接口与预期行为。
4
SCAFFOLD · 项目脚手架
生成配置文件、main.py、数据库基类等项目骨架。
5
MODULES · 模块代码
逐模块生成后端代码，按 schemas → models → dao → services → routers 顺序。
6
FRONTEND · 前端模块
逐模块生成 React 前端页面与组件，与后端契约对齐。
7
INTEGRATE · 集成装配
注册路由、生成 docker-compose.yml、__init__.py 等集成文件。
